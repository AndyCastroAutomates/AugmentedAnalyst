# Deliverable: Executive Brief

**Trigger:** Always generated.
**Audience:** Senior leadership, commanders, executives, board-level stakeholders.
**Purpose:** Under one page. Decision-forcing. Three questions answered.

---

## Format Rules

- Under one page. If it runs over, cut.
- No ATT&CK IDs, no Sigma, no JSON, no technical jargon.
- Decision Points are mandatory: 2 minimum, 4 maximum. Each must be specific, time-bound, owner-implied.
- Geopolitical context: include only when source contains explicit geopolitical framing.
- Reviewer signature line mandatory.

---

## Format

CYBER THREAT EXECUTIVE BRIEF
TLP:[marking] | [Date] | Source: [Report citation]
Reviewed by: ______________________________

**Situation**
[2-3 sentences. What happened, who is behind it, what it does to victims.]

**Who Is at Risk**
[Specific targeting from source.]

**Decision Points**
[2-4 numbered decisions. Each specific, time-bound, owner-implied.]

**Risk Assessment**
[2-3 sentences. Sophistication, likelihood, potential impact.]

**Geopolitical Context**
[Omit if not in source.]

*This brief is based solely on [source]. All claims traceable to source.*

---

## Good Decision Points Examples
- "Authorize an immediate IOC sweep. Security teams have the file hashes and indicators to determine whether this threat is present. Completable within 72 hours."
- "Restrict executable file transfers via messaging apps. This closes the confirmed delivery vector and requires a policy decision."
