# CTI Fusion Analyst

A Claude skill for cyber threat intelligence practitioners. Paste or upload a raw threat report and receive a complete suite of structured defensive artifacts in seconds.

Built and released by an independent security researcher for the CTI and security operations community.

---

## Setup

### Option A -- claude.ai

1. Open a new conversation at claude.ai.
2. Open `SKILL.md` from this folder and copy the full contents.
3. Paste at the start of your conversation.
4. Paste or upload your threat report.

### Option B -- Claude for Teams or Enterprise

1. Upload `SKILL.md` to your organization's skill library via the Admin console.
2. The skill activates automatically when threat intelligence content is detected.

### Option C -- Claude API

Use the contents of `SKILL.md` as your system prompt. Pass the source report as the user message.

---

## Usage

1. Start a new conversation.
2. Paste the skill (Option A) or let it auto-activate (Option B).
3. Paste or upload the full threat report text.
. Stage 1 Core Analysis is generated first -- review it before requesting deliverables.
5. Request deliverables: "Generate the Threat Assessment", "Generate the Executive Brief", etc.

Tips:
- Paste the full report. Thin input produces thin output.
- For PDFs: extract text first, then paste.
- Multiple related reports (same actor or campaign): paste together -- the skill merges them.
- Multiple unrelated reports: the skill will ask how to proceed before generating anything.

## Deliverables

| Deliverable | Audience | Trigger |
|-------------|----------|---------|
| Core Analysis | CTI analyst (review gate) | Always, first |
| Threat Assessment | CTI team, operational leadership | Always |
| Executive Brief | Senior leadership | Always |
| Risk Mitigation Plan | All teams | Always |
| Threat Hunt Hypothesis | Hunt and IR teams | When behavioral TTPs are huntable |
| Red Team Emulation Plan | Offensive and purple teams | When TTP fidelity supports emulation |
| Patch Notification | IT, system owners, patch management | When CVEs are present |

## Security Disclaimer

This tool assists qualified security professionals in processing threat intelligence. It is not a replacement for human analyst judgment.

Before using outputs:
- Review all ATT&CK mappings against the source document.
- Validate detection rules in a test environment before deploying to production.
- Do not deploy AI-generated Sigma rules directly to a production SIEM or EDR without review by a qualified detection engineer.
- Do not execute red team emulation steps without proper written authorization.
- Treat all IOCs as requiring independent verification before actioning.
